ADAMOperating System Setup

Operator Console
Sign in

This page talks to your local server with your bearer token (the same ADAM_TOKEN the Operator Console uses). It is stored on this device only.

Easiest way in: on your computer, start Adam with the Start Adam launcher (scripts\start-adam.ps1) — it opens you already signed in, so you won't see this box. On a phone, open the Operator Console's Connect phone and scan the sign-in QR. To copy the token by hand, run scripts\copy-token.ps1 and paste it above.

The Operating System is opt-in and stays disabled until you finish setup. It runs entirely in your own Google account, in a Google Sheet you create. Adam never holds your Google password. It can read your board, and write the XP, quests, and stats you approve. It cannot delete rows or tabs through the bridge.
Set up your Operating System

  1. Create a blank Google Sheet

    Make a new, empty spreadsheet in your own Google Drive — name it anything (e.g. "Operating System").

    Create a blank Sheet ↗

  2. Open Apps Script and copy the Operating System script

    In your new Sheet: Extensions → Apps Script. Click anywhere in the big code area (the file is Code.gs), press Ctrl+A to select everything, Delete, then come back here and copy the script:

    Back in the editor, press Ctrl+V to paste it. Nothing secret is in it.

  3. Save the project — this is what makes the functions appear

    Nothing runs until you save. In the toolbar just above the code, click the floppy-disk Save icon (💾 Save project) or press Ctrl+S.

    • If it asks you to name the project, type anything (e.g. Operating System) and save.
    • The function names only show up in the picker (next step) after you save.

  4. Run setupHunterDashboard to build the board

    In the toolbar above the code there's a function picker — a dropdown showing a function name. Click it and choose setupHunterDashboard, then click the ▶ Run button next to it. It builds every tab and formula.

    The first run asks you to authorize your own script. Follow these exact clicks:

    • A box says "Authorization required" → click Review permissions.
    • Pick your Google account.
    • On the "Google hasn't verified this app" screen, click the small grey Advanced link at the bottom-left.
    • Click Go to (your project name) (unsafe) — it's your own script, so it's safe.
    • Scroll down and click Continue / Allow at the bottom-right.

  5. Run setHunterToken to generate your token

    Open the function picker again and this time choose a different function — setHunterToken — then click ▶ Run.

    The Execution log opens at the bottom and prints a line like HUNTER_TOKEN=abc123…. Copy everything after the = sign — that long value is your bridge token. (Adam saves it for you below — you don't edit any file by hand.)

  6. Deploy as a Web app

    In the top-right corner there's a blue Deploy button. Click it → New deployment.

    • Next to "Select type" there's a small gear icon (⚙). Click it and choose Web app.
    • Set Execute as: Me and Who has access: Anyone.
    • Click Deploy. If it asks you to authorize access again, go through the same Review permissions → Advanced → Go to (unsafe) → Continue steps once more.

  7. Copy the Web app URL

    After deploying, Apps Script shows two links. Copy the second one — the Web app URL (it ends in /exec). Use the Copy button next to it.

  8. Paste your bridge URL and token here

    Your token is sent only to your own bridge. When you click Enable Operating System below, Adam saves it to your local .env for you — you never edit a file by hand. It is never put in a link and never logged.

  9. Test the connection

✓ Verified — turn it on

Your bridge is live and your token works. One click and Adam writes its own settings for you — no editing files, no copying JSON.

Prefer to set it up by hand?

You don't need to do any of this — the button above does it for you. But if you'd rather edit the files yourself:

1. Add your token to .env:

  • Open File Explorer and go to your Adam folder (C:\Users\you\adam-local).
  • Find the file named .env and double-click it. If Windows asks how to open it, choose Notepad.
  • Scroll to the Hunter dashboard bridge line and put your token right after the = so it reads HUNTER_TOKEN=yourtoken.
  • Click File → Save, then close Notepad.
