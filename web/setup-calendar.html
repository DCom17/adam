Operator Console
Sign in

This page talks to your local server with your bearer token (the same JARVIS_TOKEN the Operator Console uses). It is stored on this device only.

Easiest way in: on your computer, start Jarvis with the Start Jarvis launcher (scripts\start-jarvis.ps1) — it opens you already signed in, so you won't see this box. On a phone, open the Operator Console's Connect phone and scan the sign-in QR. To copy the token by hand, run scripts\copy-token.ps1 and paste it above.

This calendar integration is opt-in and stays disabled until you finish setup. It runs entirely in your own Google account through a small script you deploy — Jarvis never holds your Google password. It can read and create events, and edit an event after you confirm. It cannot delete events.
Set up your calendar bridge

  1. Copy the bridge code

    This is the small script that runs in your Google account. Nothing secret is in it.

  2. Open Google Apps Script

    Open a new Apps Script project ↗

  3. Paste the code

    Click anywhere in the big code area (the file is called Code.gs), press Ctrl+A to select everything, press Delete, then Ctrl+V to paste the bridge code you copied.

  4. Save the project — this is what makes the function appear

    Nothing will work until you save. Look at the toolbar just above the code: click the floppy-disk Save icon (💾 Save project), or just press Ctrl+S.

    • If Apps Script asks you to name the project, type anything (e.g. Jarvis Calendar) and save.
    • After saving, the function name shows up in the function picker in the next step. If it's greyed out, you haven't saved yet.

  5. Run setCalendarSyncToken to generate your token

    In that same toolbar above the code there's a function picker — a dropdown that currently shows a function name. Click it and choose setCalendarSyncToken. Then click the ▶ Run button right next to it.

    The first time you run it, Google asks you to authorize your own script. Follow these exact clicks:

    • A box says "Authorization required" → click Review permissions.
    • Pick your Google account.
    • You'll see a "Google hasn't verified this app" screen. At the bottom-left, click the small grey Advanced link.
    • Then click Go to (your project name) (unsafe) — this is your own script, so it's safe.
    • Scroll down and click Continue / Allow at the bottom-right.

    When it finishes, the Execution log opens at the bottom of the screen with a line like GOOGLE_CALENDAR_TOKEN=abc123…. Copy just the long value after the = sign — that's your bridge token. You'll paste it below in a moment (Jarvis saves it for you — you do not need to edit any file by hand).

  6. Deploy as a Web app

    In the top-right corner there's a blue Deploy button. Click it → New deployment.

    • Next to "Select type" there's a small gear icon (⚙). Click it and choose Web app.
    • Set Execute as: Me.
    • Set Who has access: Anyone.
    • Click Deploy (and authorize again if asked, same as above).

  7. Copy the Web app URL

    After deploying, Apps Script shows two links. Copy the second one — the Web app URL (it ends in /exec). Use the Copy button next to it.

  8. Paste your bridge URL and token here

    Your token is sent only to your own bridge. When you click Enable calendar below, Jarvis saves it to your local .env for you — you never edit a file by hand. It is never put in a link and never logged.

  9. Test the connection

✓ Verified — turn it on

Your bridge is live and your token works. One click and Jarvis writes its own settings for you — no editing files, no copying JSON.

Prefer to set it up by hand?

You don't need to do any of this — the button above does it for you. But if you'd rather edit the files yourself:

1. Add your token to .env:

  • Open File Explorer and go to your Jarvis folder (C:\Users\you\jarvis-voice-local).
  • Find the file named .env (about ten items down, it's just called .env) and double-click it. If Windows asks how to open it, choose Notepad.
  • Scroll down to the Google Calendar bridge line, and put your token right after the = so it reads GOOGLE_CALENDAR_TOKEN=yourtoken.
  • Click File → Save, then close Notepad.
