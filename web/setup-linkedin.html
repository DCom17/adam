Operator Console
Sign in

This page talks to your local server with your bearer token (the same ADAM_TOKEN the Operator Console uses). It is stored on this device only.

Easiest way in: on your computer, start Adam with the Start Adam launcher (scripts\start-adam.ps1) — it opens you already signed in, so you won't see this box. On a phone, open the Operator Console's Connect phone and scan the sign-in QR. To copy the token by hand, run scripts\copy-token.ps1 and paste it above.

This LinkedIn integration is opt-in and stays disabled until you turn it on. By default it is a drafting tool: Adam writes posts (and profile text) that you review and paste into LinkedIn by hand — no LinkedIn login, no automation. Adam cannot read your LinkedIn feed and cannot delete anything. Adam never creates or signs up for a LinkedIn account — automating signup violates LinkedIn's User Agreement.
Lane 1 — Draft posts (default, recommended)

This is the whole product for most people. With the connector on, Adam drafts a post or profile blurb, you approve it like any other draft, and you copy it into LinkedIn yourself.

  • No LinkedIn credentials needed — nothing to connect.
  • Every draft routes through the normal approval flow before you see it.
  • You paste the final text into LinkedIn by hand, so you always have the last word.

To enable just the draft lane, save this in settings.json and restart Adam:

"integrations": {
  "linkedin": {
    "enabled": true,
    "api_enabled": false,
    "client_id": "",
    "author_urn": "",
    "timeout_seconds": 20
  }
}
Saved. Draft lane is on. Restart Adam (close the black "Adam" window and relaunch), then ask Adam to "draft a LinkedIn post about …".
Lane 2 — Auto-post to your own feed (advanced, optional)
Advanced and entirely optional. This lets Adam publish an already-approved post to your own LinkedIn feed over the API. It is never automatic — every post is an explicit action you take on a draft you already approved. It needs your own LinkedIn Developer App and a member access token, which expires about every 60 days (LinkedIn issues no long-lived token), so you will re-paste it periodically. There is no shared Adam app — it must be fully your own. Adam still cannot read your feed and cannot delete posts.
Set up the API auto-post lane

  1. Create a LinkedIn Company Page

    A Developer App must be tied to a Page. Make one (it can be a personal brand page) at linkedin.com/company/setup/new ↗.

  2. Create a Developer App

    At the LinkedIn developer portal ↗, create an app and associate it with your Page.

  3. Add the products you need

    On the app's Products tab, request Share on LinkedIn and Sign In with LinkedIn using OpenID Connect. These grant the w_member_social (post) and openid/profile (identity) scopes.

  4. Run the OAuth flow and get an access token

    From the app's Auth tab, do the 3-legged OAuth flow as yourself and copy the resulting member access token. (For v1 you run this once by hand — Adam does not host a redirect. The token lasts about 60 days; re-paste it when it expires.)

    You'll also see your Client ID and Client Secret on the Auth tab.

  5. Paste your credentials here to test

    For the test, only the access token is sent — to LinkedIn, to confirm it works and resolve your author id (your Client Secret is not sent by the test). When you later click Enable auto-post lane, Adam saves the token (and Client Secret, if you entered one) to your local .env for you — you never edit a file by hand. Nothing is ever put in a link or logged.

  6. Test the connection

✓ Verified — turn on the auto-post lane

Your token works and your author id is resolved. One click and Adam writes its own settings and saves your token (and client secret, if you entered one) for you — no editing files.

Your Client Secret and access token are saved to your local .env only — never shown, linked, or logged. The token expires about every 60 days; re-run this when posting starts failing. Posting is still never automatic — every post is an explicit action on a draft you already approved.
